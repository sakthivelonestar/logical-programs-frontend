What: In Angular, Routing & Navigation is the mechanism that allows users to move between different views or components in a single-page application using the RouterModule with advanced features like lazy loading.

Why: It enables building SPA (Single Page Applications) with code splitting and lazy loading, reducing initial bundle size and improving performance by loading modules on-demand.

Where: Essential for large-scale applications, enterprise dashboards, e-commerce platforms, and any app requiring optimized loading and structured navigation.

üìå Main App Routing Module (app-routing.module.ts)


import { NgModule } from '@angular/core';
import { RouterModule, Routes, PreloadAllModules } from '@angular/router';
import { HomeComponent } from './home/home.component';
import { PageNotFoundComponent } from './page-not-found/page-not-found.component';
import { AuthGuard } from './guards/auth.guard';

const routes: Routes = [
  { path: '', component: HomeComponent },
  { 
    path: 'products', 
    loadChildren: () => import('./products/products.module').then(m => m.ProductsModule),
    canLoad: [AuthGuard]
  },
  { 
    path: 'users', 
    loadChildren: () => import('./users/users.module').then(m => m.UsersModule)
  },
  { 
    path: 'admin', 
    loadChildren: () => import('./admin/admin.module').then(m => m.AdminModule),
    canLoad: [AuthGuard],
    data: { preload: true }
  },
  { path: '404', component: PageNotFoundComponent },
  { path: '**', redirectTo: '/404' }
];

@NgModule({
  imports: [RouterModule.forRoot(routes, {
    preloadingStrategy: PreloadAllModules,
    enableTracing: false // Set to true for debugging
  })],
  exports: [RouterModule]
})
export class AppRoutingModule { }

‚öôÔ∏è Advanced Programmatic Navigation


import { Component, OnInit } from '@angular/core';
import { Router, ActivatedRoute, NavigationEnd } from '@angular/router';
import { filter } from 'rxjs/operators';

@Component({
  selector: 'app-navigation',
  template: `
    <div class="navigation-controls">
      <button (click)="goToProduct(123)">Go to Product 123</button>
      <button (click)="goToProductEdit(123)">Edit Product 123</button>
      <button (click)="goWithQueryParams()">Search Products</button>
      <button (click)="goBack()">Go Back</button>
    </div>
  `
})
export class NavigationComponent implements OnInit {
  
  constructor(
    private router: Router,
    private route: ActivatedRoute
  ) {}

  ngOnInit() {
    // Listen to route changes
    this.router.events
      .pipe(filter(event => event instanceof NavigationEnd))
      .subscribe((event: NavigationEnd) => {
        console.log('Navigated to:', event.url);
      });
  }

  // Navigate with route parameters
  goToProduct(id: number) {
    this.router.navigate(['/products', id]);
  }

  // Navigate with nested routes
  goToProductEdit(id: number) {
    this.router.navigate(['/products', id, 'edit']);
  }

  // Navigate with query parameters
  goWithQueryParams() {
    this.router.navigate(['/products'], {
      queryParams: { category: 'electronics', page: 1 },
      fragment: 'top'
    });
  }

  // Navigate relative to current route
  goBack() {
    this.router.navigate(['../'], { relativeTo: this.route });
  }

  // Navigate with state
  goWithState() {
    this.router.navigate(['/products'], {
      state: { data: { fromComponent: 'navigation' } }
    });
  }
}

üöÄ Custom Preloading Strategy


// custom-preloading.strategy.ts
import { Injectable } from '@angular/core';
import { PreloadingStrategy, Route } from '@angular/router';
import { Observable, of } from 'rxjs';

@Injectable({ providedIn: 'root' })
export class CustomPreloadingStrategy implements PreloadingStrategy {
  
  preload(route: Route, load: () => Observable<any>): Observable<any> {
    // Only preload routes marked with preload: true
    if (route.data && route.data['preload']) {
      console.log('Preloading module: ' + route.path);
      return load();
    }
    return of(null);
  }
}

// Usage in app-routing.module.ts
imports: [RouterModule.forRoot(routes, {
  preloadingStrategy: CustomPreloadingStrategy
})]

export class ProductDetailComponent implements OnInit, OnDestroy {

  page: number;

    private route: ActivatedRoute,
    private router: Router
  ) {}

    // Get route parameters
    this.route.params
      .pipe(takeUntil(this.destroy$))
      .subscribe(params => {
        const id = +params['id'];
        this.loadProduct(id);
      });

      .pipe(takeUntil(this.destroy$))

      .pipe(takeUntil(this.destroy$))

      });

20 Advanced Interview Questions on Routing & Navigation

  1. What is lazy loading and how do you implement it in Angular?

    Lazy loading loads feature modules only when needed using loadChildren with dynamic imports: loadChildren: () => import('./module').then(m => m.Module)

    Performance
  2. What's the difference between canLoad and canActivate?

    canLoad prevents loading the module, while canActivate allows loading but prevents activation. Use canLoad for lazy modules.

    Guards
  3. How do you implement a custom preloading strategy?

    Create a service implementing PreloadingStrategy interface with a preload() method that returns an Observable.

    Optimization
  4. What are auxiliary routes in Angular?

    Secondary routes displayed in named router outlets, useful for sidebar navigation or popup content alongside main content.

    Advanced
  5. How do you handle route data with Resolvers?

    Create a service implementing Resolve interface to pre-fetch data before route activation, ensuring data is available when component loads.

    Data
  6. What is the difference between navigate() and navigateByUrl()?

    navigate() takes an array of route segments, while navigateByUrl() takes a complete URL string.

    Navigation
  7. How do you implement route animations in Angular?

    Use Angular Animations API with route data and @routeAnimation trigger on the router outlet.

    Animation
  8. What are router events and how do you use them?

    Events like NavigationStart, NavigationEnd emitted during navigation, useful for loading indicators and navigation tracking.

    Events
  9. How do you implement breadcrumbs using Angular Router?

    Use route data configuration and ActivatedRoute to build breadcrumb navigation dynamically from route hierarchy.

    UX
  10. What is the purpose of pathMatch in routing?

    pathMatch: 'full' matches the entire URL, while 'prefix' (default) matches if URL starts with the path.

    Configuration
  11. How do you pass state during navigation?

    Use router.navigate() with state option: { state: { data: value } } and access via history.state.

    State
  12. What are route guards and name all types?

    Guards control access to routes: CanActivate, CanActivateChild, CanDeactivate, CanLoad, and Resolve.

    Security
  13. How do you handle lazy loading with shared modules?

    Import shared modules in each lazy-loaded feature module, or use a core module pattern with forRoot()/forChild().

    Architecture
  14. What is the difference between matrix parameters and query parameters?

    Matrix params are part of the URL segment (;key=value), while query params follow ? and are optional for route matching.

    Params
  15. How do you implement route-based feature flags?

    Use route guards or resolver services to check feature flags and conditionally allow access to routes or redirect to alternatives.

    Features
  16. What is router outlet context and how to use it?

    Router outlet exposes route context through template reference variables, allowing parent components to interact with routed child components.

    Context
  17. How do you implement route-level error handling?

    Use ErrorHandler service, route resolvers that handle errors, and navigation error events from router.

    Error Handling
  18. What is the difference between router state and activated route?

    RouterState represents the entire route tree, while ActivatedRoute represents the currently active route with its specific data.

    State
  19. How do you optimize routing for better SEO?

    Use server-side rendering (SSR), proper meta tags via Title and Meta services, and structured URLs with meaningful paths.

    SEO
  20. How do you test Angular routing and navigation?

    Use RouterTestingModule, spy on router methods, test route guards with TestBed, and verify navigation with Location service.

    Testing
