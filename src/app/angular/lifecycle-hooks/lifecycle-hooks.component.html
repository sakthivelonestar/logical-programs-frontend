üìã Example: Using Lifecycle Hooks

@Component({
  selector: 'app-logger',
  template: `<p>Logger works!</p>`
})
export class LoggerComponent implements OnInit, OnChanges, OnDestroy {

  constructor() {
    console.log('Constructor called');
  }

  ngOnChanges(changes: SimpleChanges) {
    console.log('ngOnChanges called', changes);
  }

  ngOnInit() {
    console.log('ngOnInit called');
  }

  ngOnDestroy() {
    console.log('ngOnDestroy called - cleanup here');
  }
}

15 Important Interview Questions on Angular Lifecycle Hooks

  1. What are Angular lifecycle hooks?

    Special methods that Angular calls at specific stages of a component/directive‚Äôs life, from creation to destruction.

    Basics
  2. Which hook is called only once in a component‚Äôs lifetime?

    ngOnInit is called once after the first ngOnChanges.

    Core
  3. When would you use ngOnDestroy?

    To perform cleanup tasks such as unsubscribing from Observables, removing event listeners, or clearing timers.

    Cleanup
  4. Difference between ngOnChanges and ngDoCheck?

    ngOnChanges triggers only when input properties change, while ngDoCheck allows custom change detection logic.

    Comparison
  5. Which hooks relate to content projection?

    ngAfterContentInit and ngAfterContentChecked.

    Content
  6. Which hooks deal with the component‚Äôs view?

    ngAfterViewInit and ngAfterViewChecked.

    View
  7. Can you use lifecycle hooks in directives?

    Yes, directives also support lifecycle hooks like ngOnInit, ngOnChanges, and ngOnDestroy.

    Directives
  8. What is the first lifecycle hook called?

    ngOnChanges (if there are input bindings), otherwise ngOnInit.

    Order
  9. What‚Äôs the role of the constructor in lifecycle?

    The constructor is called before Angular lifecycle hooks, used mainly for dependency injection setup.

    Constructor
  10. How does Angular handle hook execution order?

    Angular follows a strict sequence: constructor ‚Üí changes/init ‚Üí content ‚Üí view ‚Üí destroy.

    Order
  11. When should you avoid heavy logic inside lifecycle hooks?

    Avoid heavy logic in ngDoCheck and ngAfterViewChecked because they run frequently and may hurt performance.

    Performance
  12. Can you implement multiple lifecycle hooks in one component?

    Yes, a component can implement multiple hooks depending on requirements.

    Flexibility
  13. What hook is best for initializing API calls?

    ngOnInit is commonly used to fetch initial data when the component loads.

    API
  14. What happens if you don‚Äôt implement ngOnDestroy?

    Memory leaks may occur because subscriptions, intervals, or listeners may remain active even after the component is destroyed.

    Cleanup
  15. How do lifecycle hooks improve testing?

    Hooks provide predictable entry points to test initialization, updates, and cleanup in isolation.

    Testing
