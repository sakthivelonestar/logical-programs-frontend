What: Learn Angular Data Binding techniques — Interpolation, Property Binding, Event Binding, and Two-Way Binding — to connect the template with the component class.

Why: Data binding ensures seamless synchronization between the UI and the underlying logic, making Angular applications dynamic and interactive.

Where: Used in every Angular project — for displaying values (interpolation), binding attributes/properties, handling user actions (events), and building forms (two-way binding).

📝 Step 1 — Interpolation


<h2>Hello {{ username }}!</h2>

Component Example:


export class AppComponent {
  username = 'Sakthivel';
}

👉 Interpolation { { } } is used to display component data inside the template.

🔗 Step 2 — Property Binding


<img [src]="profilePic" [alt]="username">

Component Example:


export class AppComponent {
  profilePic = 'assets/profile.png';
  username = 'Sakthivel';
}

👉 Property binding uses [ ] to bind component properties to DOM element attributes.

🎯 Step 3 — Event Binding


<button (click)="sayHello()">Click Me</button>

Component Example:


export class AppComponent {
  sayHello() {
    alert('Hello from Angular!');
  }
}

👉 Event binding uses ( ) to listen for user events like clicks, keypress, input changes, etc.

🔄 Step 4 — Two-Way Binding


<input [(ngModel)]="username" placeholder="Enter your name">
<p>You typed: {{ username }}</p>

Component Example:


export class AppComponent {
  username = '';
}

👉 Two-way binding uses [( )] (banana in a box syntax) to keep input fields and component properties in sync.

15 Important Interview Questions on Data Binding

  1. What is data binding in Angular?

    Data binding is the mechanism that connects the component class and the template, enabling dynamic updates between UI and logic.

  2. What is interpolation in Angular?

    Interpolation uses { { } } to display component data inside the template.

  3. How does property binding differ from attribute binding?

    Property binding updates DOM properties (dynamic), while attribute binding sets HTML attributes (static).

  4. What is event binding in Angular?

    Event binding uses ( ) to respond to user actions like clicks, key presses, or mouse movements.

  5. What is two-way data binding?

    Two-way binding ([(ngModel)]) keeps component properties and form inputs in sync both ways.

  6. Which module is required for two-way data binding?

    The FormsModule must be imported in the module to use [(ngModel)].

  7. Can interpolation be used inside attributes?

    Yes, e.g., <img src="{ { imageUrl } }">, but property binding [src] is preferred.

  8. What is the difference between (click) and (ngSubmit)?

    Both are event bindings, but (ngSubmit) is used with forms, while (click) is used for button clicks.

  9. Can we use two-way binding without ngModel?

    Yes, but it requires combining property and event binding manually instead of using [(ngModel)].

  10. What is the syntax difference between one-way and two-way binding?

    One-way uses [ ] or ( ), while two-way uses [( )].

  11. Can we bind to custom component properties?

    Yes, using @Input() for property binding and @Output() for event binding.

  12. What is the "banana in a box" syntax?

    It refers to [( )], the syntax for two-way binding in Angular.

  13. Why prefer property binding over interpolation for attributes?

    Property binding ensures the DOM updates dynamically, while interpolation works only with string values.

  14. How does Angular handle change detection in bindings?

    Angular’s change detection checks component properties and updates the template when values change.

  15. Can we bind multiple events to the same element?

    Yes, you can bind multiple events like (click), (mouseover) on the same element.
