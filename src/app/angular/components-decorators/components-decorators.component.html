15 Important Interview Questions on Components & Decorators

  1. What is a Component in Angular?

    A component controls a portion of the UI. It is defined using the @Component decorator and contains a TypeScript class, HTML template, and optional styles.

  2. What does the @Component decorator do?

    The @Component decorator provides metadata such as selector, templateUrl, and styleUrls, which tells Angular how to create and use the component.

  3. What is the difference between @NgModule and @Component?

    @NgModule is used for modules (groups of components, directives, services), while @Component is used for individual UI components.

  4. What is @Injectable used for?

    @Injectable marks a service as available for Dependency Injection so it can be provided in modules or components.

  5. How do @Input and @Output decorators work?

    @Input allows data flow from parent to child, while @Output uses EventEmitter to send data from child to parent.

  6. What is the difference between @Directive and @Component?

    @Directive creates custom attributes/behaviors without a template, while @Component is a directive with a template.

  7. What is @Pipe decorator used for?

    It defines a custom pipe that transforms displayed data, e.g., date formatting, string manipulation.

  8. What is @HostListener?

    @HostListener listens for DOM events on the host element and triggers a method inside the directive/component.

  9. What is @HostBinding?

    @HostBinding binds a property in the directive/component to a host element property or class.

  10. What are lifecycle hooks in Components?

    Lifecycle hooks like ngOnInit, ngOnChanges, ngOnDestroy allow execution of code at specific stages of a componentâ€™s life.

  11. Can we use multiple decorators on the same class?

    Yes, but typically one main decorator is used per class, e.g., @Component for components, @Injectable for services.

  12. What is a selector in @Component?

    The selector defines the custom HTML tag used to embed the component in templates.

  13. How does Angular detect changes in @Input properties?

    Angular uses its change detection mechanism to track updates in bound properties, triggering re-rendering as needed.

  14. What is the role of metadata in decorators?

    Metadata tells Angular how to process a class, e.g., what template to use for a component or what services to inject in a module.

  15. Give an example of a custom decorator in Angular.

    You can create a custom decorator using TypeScript functions to modify classes, e.g., logging class creation or enforcing property rules.
