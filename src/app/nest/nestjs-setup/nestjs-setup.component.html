ğŸ“ NestJS Setup and Structure Interview Questions

Project Setup Questions

  • Q: What are the prerequisites and steps to create a new NestJS project?

    Prerequisites:
    • Node.js (version 12 or higher)
    • npm or yarn package manager
    Steps:
    • Install NestJS CLI: npm i -g @nestjs/cli
    • Create project: nest new my-nestjs-app
    • Choose package manager: npm or yarn
    • Navigate to project: cd my-nestjs-app
    • Start development: npm run start:dev

  • Q: Explain the key configuration files in a NestJS project.

    • nest-cli.json: CLI behavior, project structure, compilation options
    • tsconfig.json: TypeScript compiler options and paths
    • package.json: Dependencies and npm scripts
    • .env: Environment variables configuration
    • main.ts: Application bootstrap configuration

Architecture & Structure Questions

  • Q: What are the core architectural concepts in NestJS?

    • Modules: Organize application structure
    • Controllers: Handle HTTP requests and define routes
    • Providers: Services, repositories, factories, helpers
    • Middleware: Process requests before reaching route handler
    • Guards: Handle authentication and authorization
    • Interceptors: Transform request/response data
    • Pipes: Validate and transform input data

  • Q: How would you structure a large-scale NestJS application?

    • src/modules/: Feature modules (users, auth, products)
    • src/common/: Shared code (guards, filters, decorators)
    • src/config/: Configuration modules and env setup
    • src/database/: Database entities and migrations
    • src/utils/: Helper functions and utilities
    • test/: Unit and e2e tests

Development & Testing Questions

  • Q: What are the essential NestJS CLI commands for development?

    • nest g resource [name] - Generate full CRUD resource
    • nest g module [name] - Create new module
    • nest g controller [name] - Generate controller
    • nest g service [name] - Create service
    • nest g middleware [name] - Generate middleware
    • nest g guard [name] - Create guard

  • Q: How do you handle testing in a NestJS application?

    • Unit Tests: npm run test - Test individual components
    • E2E Tests: npm run test:e2e - Test full request flow
    • Test Watch: npm run test:watch - Development testing
    • Coverage: npm run test:cov - Test coverage report
