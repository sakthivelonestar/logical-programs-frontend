What: In JavaScript, Prototypes & Inheritance allow objects to inherit properties and methods from other objects, enabling code reuse and dynamic behavior.

Why: They help you create efficient object hierarchies, avoid repetition, and extend functionality across multiple objects without rewriting code.

Where: You can use them in object-oriented programming, library development, frameworks, or any project where shared behavior between objects is needed.

1. Understanding Prototypes

Every JavaScript object has an internal link to another object called its prototype. Properties and methods can be inherited from the prototype.

Basic Prototype Example

// Create an object with a method
const animal = {
  eat() {
    console.log(`${this.name} is eating`);
  }
};

// Create a new object inheriting from animal
const dog = Object.create(animal);
dog.name = "Buddy";
dog.eat();  // Buddy is eating

Key Prototype Concepts

  • Object.prototype: The root prototype from which all objects inherit.
  • __proto__: Internal property linking to the prototype.
  • Object.create(): Creates an object with a specified prototype.

2. Constructor Functions & Prototypes

Constructor functions allow you to create multiple similar objects, sharing methods via the prototype.

Example: Constructor & Prototype

// Constructor function
function Person(name, age) {
  this.name = name;
  this.age = age;
}

// Adding method to prototype
Person.prototype.greet = function() {
  console.log(`Hello, my name is ${this.name}`);
};

const alice = new Person("Alice", 25);
alice.greet();  // Hello, my name is Alice

Key Points

  • Methods defined on the prototype are shared by all instances.
  • Reduces memory usage since functions are not duplicated.
  • Supports inheritance chain through the prototype link.

3. ES6 Classes & Inheritance

Modern JavaScript uses classes to simplify constructor and prototype syntax while still using prototypes under the hood.

Example: Class Inheritance

// Base class
class Animal {
  constructor(name) {
    this.name = name;
  }

  eat() {
    console.log(`${this.name} is eating`);
  }
}

// Derived class
class Dog extends Animal {
  bark() {
    console.log(`${this.name} is barking`);
  }
}

const max = new Dog("Max");
max.eat();  // Max is eating
max.bark(); // Max is barking

Key Points

  • extends: Sets up prototype inheritance between classes.
  • super(): Calls the parent class constructor.
  • Class syntax is syntactic sugar over JavaScript's prototype-based inheritance.

4. Practical Use Cases

Common scenarios for prototypes and inheritance in JavaScript:

Shared Methods

Define methods once in prototype for all objects:


function Vehicle(type) {
  this.type = type;
}

Vehicle.prototype.move = function() {
  console.log(`${this.type} is moving`);
};

const car = new Vehicle("Car");
const bike = new Vehicle("Bike");
car.move();  // Car is moving
bike.move(); // Bike is moving

Library/Framework Objects

Objects in libraries like jQuery or frameworks often rely on prototype inheritance to share functionality.

Extending Built-in Objects

// Add new method to Array prototype
Array.prototype.first = function() {
  return this[0];
};

const arr = [10, 20, 30];
console.log(arr.first());  // 10

Key Points

  • Use prototypes to save memory by sharing methods.
  • Inheritance helps organize code and create hierarchies.
  • ES6 classes make inheritance syntax cleaner but still use prototypes internally.
